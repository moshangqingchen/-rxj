# AI 电商与外卖平台出图工作台

本项目用于通过 AI 生成和编辑不同类型的电商图、外卖平台图和营销素材图。它不是单个商品项目，也不是素材管理仓库；核心目标是把“出图需求 -> 平台规则 -> 风格选择 -> 高转化提示词 -> 多版本生成 -> 落盘验收”变成一套可重复使用的工作流。

## 适用图位

| 类型 | 常见图位 | 目标 |
| --- | --- | --- |
| 电商商品图 | 主图、轮播图、白底图、SKU 图、场景图 | 提高点击率、突出商品、减少审核风险 |
| 电商详情页 | 首屏、卖点、参数、场景、细节、售后 | 建立信任、讲清卖点、促进转化 |
| 店铺与活动图 | 店铺 Banner、活动海报、促销入口图 | 强化活动主题、保证多端裁切安全 |
| 外卖平台图 | 菜品图、套餐图、分类图、门店图、活动图 | 看得清、像实物、有食欲、不过度误导 |
| 跨境平台图 | Amazon 主图、副图、A+ 模块，Shopee/Lazada/Alibaba 商品图 | 符合白底/尺寸/语言/站点差异要求 |

## 核心规则

1. 用户提出“生图、画一张、生成图片、做一张图、改图、图片编辑”等需求时，默认直接调用 image2.0 / image generation 工具，不只给提示词。
2. 图片里的文字、二维码、版面元素和最终画面都由当前可用的 image generation 模型生成或编辑，除非用户明确要求只写提示词、SVG、HTML/CSS 或代码资产。
3. 当需求包含“主图、商品主图、电商主图、精美主图”时，默认按高转化电商主图方向处理：产品突出、卖点清晰、标题有冲击力、标签和角标增强购买理由。
4. AI 生成或编辑后的最终图片必须保存到本地文件。若用户没有指定目录，按任务名称在当前工作区新建文件夹，并使用清晰、连续的文件名。
5. 平台尺寸、上传限制和审核规则会变化，发布前以对应平台后台实时提示为准。

## 推荐工作流

1. 用 [出图需求模板.md](出图需求模板.md) 补齐商品、平台、图位、尺寸、卖点、文案和禁忌。
2. 查 [平台图位速查表.md](平台图位速查表.md)，确认图位比例、常用尺寸、风格重点和审核禁忌。
3. 查 [多风格设计库.md](多风格设计库.md)，为同一商品选择 3 到 5 个差异明显的视觉方向。
4. 用 [平台风格包.md](平台风格包.md) 和 [品类默认风格组合.md](品类默认风格组合.md) 选择默认风格配方。
5. 用 [趋势跟进与多版本出图策略.md](趋势跟进与多版本出图策略.md) 判断是否要加入当季潮流、平台人群风格或 A/B 测试版本。
6. 按 [多风格趋势化出图执行方案.md](多风格趋势化出图执行方案.md) 决定生成 1、3 还是 5 个版本。
7. 从 [提示词模板](提示词模板) 里选择合适模板，按品类、平台和风格改写。
8. 直接调用 image generation 生成或编辑图片，成品落盘到任务文件夹。
9. 用 [生成质量检查清单.md](生成质量检查清单.md) 验收文字、商品真实性、平台合规、尺寸和可读性。
10. 用 [生成记录模板.md](生成记录模板.md) 记录提示词、版本、文件路径和验收结论。
11. 若需要强化转化文案，从 [文案与卖点模板.md](文案与卖点模板.md) 选择标题、标签、角标和 CTA 结构。

## 建议任务目录

```text
任务名称/
  00_需求.md
  01_生成记录.md
  02_初稿/
  03_精选/
  04_最终交付/
```

目录只是建议，不要求把项目变成素材管理库。它的作用是让每次 AI 出图能保留需求、提示词和最终交付路径，方便复盘和改版。
