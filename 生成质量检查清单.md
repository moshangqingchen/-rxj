# 生成质量检查清单

用于判断 AI 生成的电商图、外卖图是否可继续优化、可交付、可上传。每次生成后至少做一次缩略图检查和文字检查。

## 通用检查

| 检查项 | 通过标准 |
| --- | --- |
| 文件已落盘 | 最终图已保存到本地任务目录，并告知保存路径 |
| 尺寸 | 与目标图位一致，或有清楚的二次裁切/导出版本 |
| 格式 | JPG/PNG 等符合平台要求 |
| 主体 | 商品或菜品一眼可识别，缩小到 120x120 仍能看清类型 |
| 清晰度 | 无糊、无脏污、无主体边缘破碎 |
| 构图 | 重要元素在安全区内，未被裁切或贴边 |
| 文字 | 无错别字、乱码、随机字符、英文占位、截断和重影 |
| 信息真实性 | 数量、规格、颜色、包装、配件与实际售卖一致 |
| 合规 | 无二维码、联系方式、外链引导、竞品商标、未授权 Logo |
| 文案风险 | 无极限词、医疗功效、绝对化承诺、虚假前后对比 |

## 电商主图

| 检查项 | 通过标准 |
| --- | --- |
| 点击吸引力 | 主标题清楚，卖点直接，产品占比足够 |
| 主体占比 | 国内主图建议 70%-85%；Amazon 主图约 85% 以上 |
| 卖点数量 | 首图不堆字，通常 1 个标题 + 2 到 4 个标签 |
| 白底要求 | 白底图必须无文字、无促销、无场景、无重阴影 |
| 促销感 | 平台允许时可有角标/标签，但不能遮挡商品 |
| SKU 一致 | 颜色、规格、套装数量和后台选项一致 |

## 详情页

| 检查项 | 通过标准 |
| --- | --- |
| 模块顺序 | 首屏 -> 核心卖点 -> 细节/参数 -> 场景 -> 保障 |
| 单屏重点 | 每屏只讲一个主卖点，避免小字密集 |
| 移动可读 | 正文不小于 24px 感知，核心标题足够醒目 |
| 参数准确 | 尺寸、重量、容量、材质、保存方式不编造 |
| 证明材料 | 证书、报告、认证等不虚构，不使用无法验证的标识 |
| 切片合理 | 单张高度建议 1000 到 1500，加载和维护更稳定 |

## 外卖菜品图

| 检查项 | 通过标准 |
| --- | --- |
| 真实感 | 份量、配菜、餐盒、摆盘与实际出餐基本一致 |
| 食欲感 | 光线明亮，颜色自然，不脏、不油腻失真 |
| 主菜占比 | 主菜占画面约 60%-80%，边缘有安全距离 |
| 文案控制 | 菜品图少字或无字，不把价格、满减、配送费写满 |
| 套餐一致 | 套餐内主食、饮品、小食、赠品与售卖内容一致 |
| 平台组件 | 不重复写平台会自动展示的价格、优惠、配送信息 |

## 活动图与 Banner

| 检查项 | 通过标准 |
| --- | --- |
| 信息层级 | 品牌/活动主题 > 核心利益点 > 时间/规则 |
| 安全区 | 关键文字距离边缘至少 8%，移动端不被按钮遮挡 |
| 多端裁切 | PC 宽屏两侧只放氛围，不放关键文字 |
| 视觉焦点 | 不同时出现多个同等级大标题 |
| 活动规则 | 不写无法兑现或容易违规的绝对承诺 |

## AI 常见错误专项

| 错误 | 处理方式 |
| --- | --- |
| 文字乱码 | 降低文案数量，改成更短更大的文字，重新生成 |
| 商品变形 | 强化“真实商品结构、无变形、无多余零件”约束 |
| 随机包装字 | 明确“包装无随机文字、无条码、无虚构 Logo” |
| 菜品份量夸张 | 加入“与实际售卖份量一致、不夸大、不堆叠误导” |
| 多出品牌 | 加入“no logo, no watermark, no trademark, no random brand” |
| 人手异常 | 尽量避免手部，或限制“only natural hands, no extra fingers” |
