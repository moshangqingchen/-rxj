# 多风格趋势化出图执行方案

本方案用于把“图的设计要多风格、跟紧潮流”落到实际 AI 出图流程里。目标不是盲目追热点，而是让同一商品在不同平台、人群和风格下都有可测试的候选图。

## 一句话原则

同一商品至少生成 3 个风格方向，重点商品生成 5 个风格方向；每个方向都保持商品真实、卖点一致、文字清楚、平台合规。

## 执行流程

| 步骤 | 动作 | 使用文件 |
| --- | --- | --- |
| 1 | 确认平台、图位、尺寸、卖点、文案、禁忌 | [出图需求模板.md](出图需求模板.md) |
| 2 | 确认平台尺寸和审核边界 | [平台图位速查表.md](平台图位速查表.md) |
| 3 | 选择平台默认风格包 | [平台风格包.md](平台风格包.md) |
| 4 | 选择品类默认风格组合 | [品类默认风格组合.md](品类默认风格组合.md) |
| 5 | 加入 1 到 2 个当季趋势方向 | [趋势跟进与多版本出图策略.md](趋势跟进与多版本出图策略.md) |
| 6 | 生成单张或批量多风格提示词 | [提示词模板/多风格主图批量生成.md](提示词模板/多风格主图批量生成.md) |
| 7 | 调用 image generation 逐张生成，保存到本地 | [AGENTS.md](AGENTS.md) |
| 8 | 按缩略图、文字、真实性、平台合规筛选 | [生成质量检查清单.md](生成质量检查清单.md) |
| 9 | 记录风格、提示词、文件路径和复盘结论 | [生成记录模板.md](生成记录模板.md) |

## 默认出图数量

| 任务级别 | 数量 | 风格结构 |
| --- | ---: | --- |
| 快速出图 | 1 | 按平台和品类最稳妥风格 |
| 常规商品 | 3 | 转化版 + 场景版 + 品质版 |
| 重点商品 | 5 | 转化版 + 场景版 + 品质版 + 潮流版 + 礼品/活动版 |
| 外卖菜品 | 2 到 3 | 真实食欲 + 门店热卖 + 套餐/夜宵 |
| Amazon 主图 | 1 到 2 | 合规白底主图 + 副图方向，不做主图潮流化 |

## 5 版标准结构

| 版本 | 名称 | 目标 | 适合判断 |
| --- | --- | --- | --- |
| A | 稳妥转化版 | 保底可用，清楚卖点 | 缩略图是否一眼看懂 |
| B | 品质高级版 | 提升信任和客单价 | 是否更有品牌感 |
| C | 场景种草版 | 让用户代入使用 | 是否更像真实推荐 |
| D | 潮流实验版 | 测年轻用户和活动点击 | 是否有新鲜感但不过度 |
| E | 礼品/活动版 | 测节日、送礼、大促 | 是否有仪式感和购买理由 |

## 风格决策树

```text
如果用户只说“做主图”
  -> 按平台 3 版包 + 品类默认组合

如果用户说“多风格”
  -> 按平台 5 版包 + 品类默认组合

如果用户说“跟潮流”
  -> 在 5 版中加入冷感科技 / Neo Deco / 复古怀旧 / Y2K / 异星未来感等 1-2 个趋势方向

如果用户说“高级”
  -> 高级极简 + 杂志大片 + Neo Deco / 新中式

如果用户说“外卖”
  -> 外卖真实食欲优先，不做虚假海报感

如果用户说“Amazon”
  -> MAIN 主图白底无字，趋势只用于副图/A+
```

## 文件命名规则

```text
{平台}_{商品}_{图位}_{风格名}_{序号}_{尺寸}_{日期}.png
```

示例：

```text
淘宝_农家甜玉米_主图_高转化爆款_01_1000x1000_20260619.png
淘宝_农家甜玉米_主图_自然产地_02_1000x1000_20260619.png
小红书_儿童学院风套装_封面_杂志大片_03_1000x1000_20260619.png
美团_黄焖鸡米饭_菜品图_外卖真实食欲_01_800x600_20260619.png
```

## 选择趋势的边界

| 可以做 | 不建议做 |
| --- | --- |
| 把趋势放进色彩、材质、背景、角标、光影 | 用趋势遮挡商品 |
| 给重点商品增加 1 到 2 个潮流实验版 | 所有图都做成同一种潮流 |
| 用冷感科技做数码/小家电 | 用冷感科技做需要热乎食欲的外卖菜 |
| 用 Neo Deco 做礼品/饰品/高客单 | 用轻奢装饰弱化低价利益点 |
| 用真实生活感做种草和私域 | 把真实感做成随便、脏乱、低清晰度 |

## 验收标准

1. 多风格版本之间有明显差异，不只是换背景颜色。
2. 商品主体、规格、数量、包装和核心卖点保持一致。
3. 文案少而清楚，没有乱码、错字、截断和随机字符。
4. 潮流元素不影响平台合规和商品识别。
5. 文件落盘，文件名标明风格，生成记录可复盘。

## 推荐默认口令

```text
按“{平台} 5 版平台风格包 + {品类}默认风格组合”，为{商品}生成{数量}张{图位}。
要求多风格、跟当前趋势，但商品真实、卖点一致、文字清楚。
每张图单独生成并保存到本地，文件名标明风格。
```
