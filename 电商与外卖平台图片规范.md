# 电商与外卖平台图片规范

更新时间：2026-06-15  
适用对象：电商运营、平面设计、电商摄影、外卖菜单维护、商品上架审核

## 使用说明

本文件整理主流电商平台和外卖平台常见图片图位、推荐尺寸、上传底线与设计禁忌。平台规则会按类目、活动、国家站点和商家后台版本变化，发布前必须以对应平台上传页的实时提示为准。

可信度标记：

| 标记 | 含义 |
| --- | --- |
| 官方公开来源 | 平台公开帮助中心、规则中心、卖家中心或开放平台能查到的规则入口。 |
| 商家后台常用口径 | 大量商家后台、代运营和设计交付中常用的规格，实际仍需按上传页复核。 |
| 待后台复核 | 平台公开信息不足，或新业务/活动图位频繁变化，必须进入后台确认。 |

全文件尺寸单位均为 `px`，写法为 `宽 x 高`。

## 全平台通用交付标准

| 项目 | 建议规范 |
| --- | --- |
| 色彩模式 | 使用 RGB、sRGB 色彩空间，避免 CMYK 直接导出上架。 |
| 图片格式 | 摄影图优先 JPG/JPEG；透明元素、Logo、图标可用 PNG；跨境平台需留意是否接受 TIFF/WebP。 |
| 分辨率 | 平台看像素，不看印刷 DPI。源文件建议按目标尺寸 1x 到 2x 制作。 |
| 主体占比 | 主图中商品主体建议占画面 70% 到 85%；亚马逊主图要求商品填满画面至少约 85%。 |
| 安全边距 | 重要文字、Logo、卖点、按钮感元素距离边缘至少 5% 到 8%；移动端底部常有价格、券、购物车遮挡。 |
| 文字控制 | 主图尽量少字，白底图不放字；详情页文字字号建议移动端不低于 24，核心标题不低于 32。 |
| 禁忌内容 | 避免平台外联系方式、二维码、夸大极限词、医疗功效承诺、虚假前后对比、竞品商标、侵权素材。 |
| 文件大小 | 国内电商常按 1MB 到 5MB 控制；跨境平台常见上限 5MB 到 10MB；详情页切片建议单张不超过 1MB 到 3MB。 |
| 命名规则 | `平台_商品名_SKU_图位_序号_尺寸_日期.jpg`，例如 `淘宝_便携榨汁杯_白色_主图_01_800x800_20260615.jpg`。 |
| 源文件分层 | 保留 `背景 / 产品 / 阴影 / 卖点文字 / 价格活动 / 平台专用遮罩 / 导出切片` 分层，方便多平台适配。 |

## 平台规范

### 淘宝

资料可信度：商家后台常用口径，规则入口为官方公开来源。  
复核重点：不同行业类目、淘宝旺铺模块、活动报名图位会给出单独尺寸。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| 主图/轮播图 | 800 x 800 | 建议 1:1，高清类目建议不低于 700 x 700 | JPG/PNG，常按 3MB 内控制 | 第一张就是搜索和列表主图。主体清晰，不加明显边框、水印、牛皮癣文案。 |
| 白底图 | 800 x 800 | 1:1 | JPG/PNG | 纯白背景，商品单主体，不放促销字、Logo、边框、场景道具。 |
| 长图主图 | 750 x 1000 或 800 x 1200 | 3:4 | JPG/PNG | 适合服饰、家居、内容化推荐。头部留安全边距，避免搜索裁切。 |
| SKU 图 | 800 x 800 | 1:1 | JPG/PNG | 每个颜色/规格单独拍摄，角度统一，避免只用色块代替实物。 |
| 详情页 | 750 x 不限高，分段切片 | 移动端优先 750 宽 | JPG 为主，单张建议 1MB 到 3MB 内 | 按卖点、参数、场景、细节、售后拆分；移动端文字字号不小于 24。 |
| 店铺装修/活动图 | PC 常用 1920 宽，移动端常用 750 宽 | 按旺铺模块 | JPG/PNG | 重点信息居中，PC 宽屏背景两侧不要放关键信息。 |

设计备注：

- 淘宝商品图可展示使用场景，但搜索主图应优先保证商品本身清楚。
- 不建议把价格、倒计时、二维码、联系方式放在主图上。
- 大促活动图以活动报名页尺寸为准，活动素材常和日常商品素材不同。

### 天猫

资料可信度：商家后台常用口径，规则入口为官方公开来源。  
复核重点：天猫活动、品牌旗舰店装修、行业会场图位会覆盖日常尺寸。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| 主图/轮播图 | 800 x 800 | 1:1 | JPG/PNG，常按 3MB 内控制 | 品牌质感优先，首图少字或无字，避免夸张促销贴片。 |
| 白底图 | 800 x 800 | 1:1 | JPG/PNG | 纯白背景，商品完整露出，不能加阴影过重、文字、边框、Logo。 |
| 场景图 | 800 x 800 或 750 x 1000 | 1:1 或 3:4 | JPG/PNG | 用于表达使用场景、材质、搭配，不要误导商品数量。 |
| SKU 图 | 800 x 800 | 1:1 | JPG/PNG | 颜色、材质、规格差异必须可辨认。 |
| PC 详情页 | 790 x 不限高，分段切片 | 790 宽常用 | JPG 为主 | PC 详情建议单独适配，不要直接拉伸移动端 750 图。 |
| 移动详情页 | 750 x 不限高，分段切片 | 750 宽常用 | JPG 为主 | 首屏先讲核心利益点，再讲参数、对比、证明、售后。 |

设计备注：

- 天猫更重品牌一致性，主图尽量统一光影、色温、角度。
- 食品、保健、美妆、母婴等类目尤其要避免绝对化、功效化表述。

### 拼多多

资料可信度：商家后台常用口径，开放平台和商家后台为官方公开入口。  
复核重点：多多场景、百亿补贴、品牌黑标、活动报名图位会有独立要求。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| 商品主图/轮播图 | 800 x 800 或 1000 x 1000 | 常见最低 480 x 480，1:1 | JPG/PNG，常按 1MB 到 3MB 控制 | 第一张为主图，商品居中，卖点尽量少而大。 |
| 白底图 | 800 x 800 | 1:1，常见最低 480 x 480 | JPG/PNG | 纯白背景，不能出现文字、Logo、边框、拼接、模特遮挡主体。 |
| 场景图 | 800 x 800 或 1200 x 1200 | 1:1 | JPG/PNG | 展示真实使用场景，避免把赠品拍得像主商品。 |
| SKU 图 | 800 x 800 | 1:1 | JPG/PNG | 多规格商品要做到颜色、数量、套装内容明确。 |
| 详情图 | 750 x 不限高，或 790 x 不限高 | 后台常见支持 480 到 1200 宽 | JPG/PNG | 单张建议切到 1000 到 1500 高，文字大，参数表不要太密。 |
| 活动/素材图 | 800 x 800、750 x 352、或后台指定 | 依活动页 | JPG/PNG | 活动图位变动频繁，必须按报名页提示导出。 |

设计备注：

- 拼多多首图通常转化压力高，主卖点要直接，但不要堆满低价贴片。
- 白底图和活动素材常被系统分发到不同频道，建议单独导出。

### 京东商城

资料可信度：商家后台常用口径，规则中心和京麦商家帮助为官方公开入口。  
复核重点：POP、自营、京东超市、秒杀、品牌活动图位可能不同。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| 主图/轮播图 | 800 x 800 或 1000 x 1000 | 1:1，建议不低于 800 x 800 | JPG/PNG，常按 3MB 到 5MB 控制 | 主体完整、清晰，首图不做复杂拼接，不加明显水印。 |
| 白底图 | 800 x 800 | 1:1 | JPG/PNG | 白底、单主体、无文字、无边框、无促销元素。 |
| 场景图 | 800 x 800 或 1000 x 1000 | 1:1 | JPG/PNG | 可表达使用场景和规格，但不得夸大功能。 |
| SKU 图 | 800 x 800 | 1:1 | JPG/PNG | 不同颜色、容量、套装要可区分。 |
| PC 详情页 | 790 x 不限高，分段切片 | 790 宽常用 | JPG 为主 | 京东 PC 详情常按 790 宽制作，单张不要过长。 |
| 移动详情页 | 750 x 不限高，分段切片 | 750 宽常用 | JPG 为主 | 移动端优先，标题、参数、证书图要清晰可读。 |
| 店铺装修/活动图 | PC 常用 1920 宽，移动端常用 750 宽 | 按京麦模块 | JPG/PNG | 活动、秒杀、优惠券图位必须按后台模板。 |

设计备注：

- 京东图像审核对主图清晰度、品牌授权、功效表述较敏感。
- 家电、数码、家装类建议增加参数图和尺寸图，但不要把参数塞进首图。

### 京东外卖

资料可信度：待后台复核。  
复核重点：京东外卖仍属于快速变化业务，菜品、门店、活动素材应以入驻后台和类目要求为准。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| 菜品图 | 800 x 600 | 4:3，建议不低于 600 x 450 | JPG/PNG，建议 5MB 内 | 实拍优先，菜品占画面 60% 到 80%，光线自然，不加大段文字。 |
| 套餐图 | 800 x 600 | 4:3 | JPG/PNG | 套餐包含物要完整出现，数量和实际售卖一致。 |
| 店铺头像/Logo | 800 x 800 | 1:1 | JPG/PNG | Logo 简洁，餐饮品牌名可读，避免过细小字。 |
| 门脸/店招图 | 800 x 450 或 750 x 300 | 16:9 或后台指定 | JPG/PNG | 展示真实门店或品牌门头，不建议用纯广告图冒充门脸。 |
| 分类图 | 800 x 800 或 800 x 600 | 1:1 或 4:3 | JPG/PNG | 用于热销、套餐、饮品、小吃等分类入口。 |
| 活动图 | 750 x 300、750 x 390、1280 x 720 | 后台指定 | JPG/PNG | 活动尺寸变化大，上传前按后台裁切预览修正。 |

设计备注：

- 外卖图要优先解决“看得清、像实物、有食欲”，不要过度修图导致投诉。
- 价格、满减、配送费等信息通常由平台组件展示，图片里不建议重复写。

### Amazon

资料可信度：官方公开来源。  
复核重点：不同站点和类目可能有补充规则，服饰、鞋靴、珠宝、成人用品等类目尤其要看类目风格指南。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| MAIN 主图 | 1600 x 1600 或 2000 x 2000 | 最长边至少 1000，最大边通常不超过 10000；1:1 最通用 | JPEG/JPG 优先，PNG/TIFF 可用；常按 10MB 内控制 | 纯白背景，商品占画面约 85% 以上，无文字、Logo、边框、水印、道具。 |
| 副图/轮播图 | 1600 x 1600 或 2000 x 2000 | 建议最长边不低于 1000 | JPEG/JPG/PNG/TIFF | 可用场景、细节、尺寸、功能、包装图，但不得使用虚假对比和违规声明。 |
| 变体图 | 1600 x 1600 | 1:1 | JPEG/JPG | 每个颜色/尺寸变体应能清楚区别，角度保持一致。 |
| A+ 标准模块 | 常用 970 x 300、970 x 600、300 x 300 | 按 A+ 模块 | JPG/PNG | A+ 内容按模块裁切，文字要少，避免直接嵌入过多小字。 |
| Brand Story/品牌故事 | 常见 1464 x 625 等模块尺寸 | 按后台模块 | JPG/PNG | 背景图要预留文字和卡片叠加区域。 |
| Amazon Store | 常见 3000 x 600、1500 x 750、1500 x 1500 | 按 Store 模块 | JPG/PNG | Store 组件裁切较多，主体居中，保留 8% 安全边距。 |

设计备注：

- 亚马逊主图不要放促销、尺码说明、卖点文案或配件未售卖暗示。
- 副图可以承载信息图，但移动端缩略图很小，卖点文字建议控制在 3 到 5 条。
- 图片命名和后台上传顺序要和 SKU 一致，减少变体错图。

### 阿里巴巴国际站 Alibaba.com

资料可信度：官方公开入口加商家后台常用口径。  
复核重点：国际站类目、RFQ 场景、广告素材和旺铺装修图位会有不同模板。

| 图位 | 推荐尺寸 | 最低/比例 | 格式与大小 | 设计要求 |
| --- | ---: | --- | --- | --- |
| 主图 | 1000 x 1000 或 1200 x 1200 | 建议不低于 750 x 750，1:1 | JPG/PNG，常按 5MB 内 | 背景干净，主体突出，适合 B2B 买家快速识别规格。 |
| 白底图 | 1000 x 1000 | 1:1 | JPG/PNG | 单主体、无文字、无水印、无复杂阴影。 |
| 场景/应用图 | 1000 x 1000 或 1200 x 900 | 1:1 或 4:3 | JPG/PNG | 展示应用场景、工厂、细节、包装、认证。 |
| SKU/规格图 | 1000 x 1000 | 1:1 | JPG/PNG | 颜色、材质、尺寸、配件要清楚，适合询盘。 |
| 详情页 | 750 x 不限高 或 1000 x 不限高 | 移动端建议 750 宽 | JPG/PNG | B2B 详情建议包含参数表、生产能力、包装、运输、证书。 |
| 旺铺/活动 Banner | 1920 x 480、1200 x 300、750 x 300 | 按旺铺模块 | JPG/PNG | 国际买家阅读习惯偏理性，避免只做大促风格。 |

设计备注：

- 国际站买家更看重规格、材质、认证、MOQ、交期、应用场景。
- 不建议在主图放大量英文卖点，可放到副图和详情页。

## 按图位制作建议

### 主图

| 检查项 | 要求 |
| --- | --- |
| 商品是否一眼可识别 | 缩小到 120 x 120 仍能看出商品类型。 |
| 主体是否足够大 | 国内平台建议 70% 到 85%，亚马逊主图约 85% 以上。 |
| 是否有违规元素 | 主图避免二维码、联系方式、平台外引导、极限词、未经授权 Logo。 |
| 背景是否干净 | 白底图必须纯白；场景主图也要避免背景抢主体。 |
| 是否和实际售卖一致 | 套装数量、赠品、颜色、容量不能误导。 |

### 详情页

| 模块顺序 | 内容建议 |
| --- | --- |
| 首屏 | 商品是什么、核心卖点、适用场景。 |
| 卖点 | 3 到 5 个核心利益点，每屏只讲一个重点。 |
| 参数 | 尺寸、重量、材质、容量、颜色、包装清单。 |
| 场景 | 使用场景、对比、安装/食用/穿搭方式。 |
| 证明 | 检测报告、认证、用户场景、工厂实力、品牌背书。 |
| 售后 | 发货、质保、退换、注意事项。 |

详情页切片建议：

- 移动端宽度统一 `750`，单张高度建议 `1000` 到 `1500`。
- 京东/天猫 PC 详情常用 `790` 宽，淘宝 PC 详情常用 `750` 宽。
- 不要把整张详情长图一次性上传，加载慢且局部修改困难。

### 外卖菜品图

| 检查项 | 要求 |
| --- | --- |
| 真实性 | 必须与实际出餐份量、配料、包装基本一致。 |
| 画面比例 | 优先 4:3 横图，通用画布 `800 x 600`。 |
| 光线 | 自然亮，避免偏色和重滤镜。 |
| 构图 | 主菜居中，边缘留 5% 到 8% 安全边距。 |
| 文案 | 少加字，不写价格、满减、配送费，避免平台组件重复。 |

### 店铺 Banner 和活动图

| 检查项 | 要求 |
| --- | --- |
| 安全区 | 左右和上下至少留 8%，移动端尤其注意按钮遮挡。 |
| 信息层级 | 品牌/活动主题 > 核心利益点 > 时间/规则。 |
| 多端适配 | PC 宽屏图两侧可做氛围，不放关键文字。 |
| 导出 | 每个平台单独导出，不要用一张图强行适配所有比例。 |

## 多平台适配工作流

1. 先做母版源文件：建议 `2000 x 2000` 方图源文件，产品、阴影、背景、文字分层。
2. 导出国内电商主图：`800 x 800`、`1000 x 1000` 两套。
3. 导出跨境主图：`1600 x 1600` 或 `2000 x 2000`，亚马逊主图单独做纯白无字版。
4. 导出内容长图：`750 x 1000`、`900 x 1200`，用于抖音、小红书、服饰类长图。
5. 导出详情页：移动端 `750` 宽，京东/天猫 PC 端另导 `790` 宽。
6. 外卖菜品单独拍摄和导出：`800 x 600`，不要直接从电商方图裁切。
7. 每次活动报名前，进入平台后台下载或查看最新模板，按模板重新裁切。
